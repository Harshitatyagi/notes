Springboot Interview Questions

✅ Spring Boot Interview Q&A Notes

1. What do you understand by Spring Boot?

Spring Boot is an extension of the Spring framework that simplifies the setup and development of new Spring applications.

It removes the need for boilerplate configuration and allows developers to get started quickly with production-ready applications.

2. What do you understand by Spring?

Spring is a Java framework that provides infrastructure support for developing Java applications.

It helps in building loosely coupled, testable, and maintainable code using Dependency Injection (DI) and Aspect-Oriented Programming (AOP).

3. Explain JAR and WAR and their related topics.

  • JAR (Java ARchive): Packages Java classes and metadata into a single file. Used for standalone applications.
  • WAR (Web Application ARchive): Includes servlets, JSPs, and other web resources. Used for deploying on external servers like Tomcat.

Spring Boot prefers JAR packaging as it embeds the server within the app (e.g., Tomcat), making deployment simpler.

5. What are the main features of Spring Boot?

  • Auto-configuration
  • Spring Boot Starters
  • Spring Boot CLI
  • Embedded servers
  • Production-ready features (via Actuator)
  • No XML configuration needed

6. Explain Auto-Configuration in simple terms.

Think of auto-configuration as Spring Boot “guessing” what you want and setting it up for you.

Example:

If spring-boot-starter-web is on the classpath, Spring Boot configures Tomcat, Jackson, MVC, etc., without you needing to write any configuration.

Layman Example:

Like opening MS Word and it automatically enables spell check, grammar, page size—based on what document type you start.

7. Give another example of auto-configuration.

If you add spring-boot-starter-data-jpa, Spring Boot:

  • Detects JPA on classpath
  • Auto-configures DataSource, EntityManagerFactory, and TransactionManager
  • Sets up connection to the database if properties are provided

8. What are Spring Boot Starters?

Starters are dependency descriptors.

They bundle common dependencies needed for a specific feature into a single dependency.

Example:

  • spring-boot-starter-web: Includes Spring MVC + embedded Tomcat + Jackson
  • spring-boot-starter-data-jpa: Includes Spring Data JPA + Hibernate

9. What is the purpose of @SpringBootApplication annotation?

It's a shortcut for:

  • @Configuration
  • @EnableAutoConfiguration
  • @ComponentScan

Used on the main class to indicate it’s the Spring Boot app’s entry point.

10. Do we use @EnableAutoConfiguration and @ComponentScan elsewhere?

No, generally you don’t. Since @SpringBootApplication includes them, using them separately is not required unless customizing behavior.

11. Explain @Configuration annotation.

Marks a class as a configuration class that contains bean definitions.

🟢 Differences from @Component:

  • Used for Java-based configuration.
  • Methods annotated with @Bean return singleton beans.
  • More powerful and explicit than component scanning.

12. Example of @Configuration vs @Component:

@Configuration
public class AppConfig {
   @Bean
   public MyService myService() {
      return new MyService();
   }
}

If you just annotate MyService with @Component, Spring will manage it but not in a centralized configuration file.

13. Role of application.properties in Spring Boot?

Used for external configuration:

  • Port: server.port=8081
  • DB config: spring.datasource.url=...
  • Logging: logging.level.org.springframework=DEBUG

14. How do you access application properties in code?

  • Using @Value("${property.name}")
  • Or by binding to a @ConfigurationProperties class

15. What is Spring Boot DevTools?

A set of tools for rapid development:

  • Auto-restart on file changes
  • Live reload in browsers
  • Disables template caching for Thymeleaf/Freemarker

16. What are Embedded Servers in Spring Boot?

Servers embedded inside the Spring Boot JAR, so no need for external deployment.

Common Embedded Servers:

  • Tomcat (default)
  • Jetty
  • Undertow

17. What is Spring Boot Actuator?

Actuator exposes production-ready endpoints like:

  • /actuator/health
  • /actuator/metrics
  • /actuator/env

    Useful for monitoring and management.

18. What is the purpose of Spring Boot Starter Parent?

Provides:

  • Dependency management
  • Plugin configuration (for Maven)
  • Default Java version, encoding, etc.

It avoids version conflicts by aligning all dependencies.

19. What does @RestController do?

Combines:

  • @Controller
  • @ResponseBody

Used to return JSON/XML directly instead of rendering a view.

20. What are @Controller and @ResponseBody?

  • @Controller: Handles web requests and returns views (like HTML).
  • @ResponseBody: Tells Spring to write the return value directly to the HTTP response body.

@RestController = @Controller + @ResponseBody

21. If I return plain text from @Controller, what happens?

Spring tries to resolve the string as a view name (like a JSP or Thymeleaf page).

Use @ResponseBody if you want to return raw text instead of a view.

22. Difference between @Component, @Service, @Repository, and @Controller?

AnnotationPurpose
@ComponentGeneric stereotype for any bean
@ServiceUsed for service/business logic layer
@RepositoryDAO layer + exception translation
@ControllerHandles web requests

All are detected via component scanning and managed by Spring's container.

23. Does @Service or @Repository behave differently from @Component?

Internally, they behave the same as @Component, but are used for semantic clarity and may enable special behavior:

  • @Repository enables automatic exception translation.
  • @Service may be used by AOP for business logic interceptors.

24. How does Spring Boot handle dependency management?

Through:

  • Spring Boot Starter Parent
  • Starters with curated dependencies
  • Avoids version mismatch via dependencyManagement block

25. Role of @Value annotation?

Injects values from:

  • application.properties
  • Environment variables
  • Command-line arguments

Example:

@Value("${app.name}")
private String appName;

26. How do you handle exceptions in Spring Boot?

  1. Using @ControllerAdvice + @ExceptionHandler
  1. Global Handling for all controllers
  1. Return custom error responses

27. Example of realistic exception handling

@ControllerAdvice
public class GlobalExceptionHandler {

   @ExceptionHandler(UserNotFoundException.class)
   public ResponseEntity<String> handleUserNotFound(UserNotFoundException ex) {
      return new ResponseEntity<>(ex.getMessage(), HttpStatus.NOT_FOUND);
   }
}

28. What is a Record in Spring Boot?

Java Records are immutable data classes introduced in Java 14+.

public record UserDTO(String name, int age) {}
  • Auto-generates constructor, getters, equals(), hashCode(), toString()
  • Ideal for DTOs

29. How does Spring Boot support externalized configuration?

Via:

  • application.properties / application.yml
  • Environment variables
  • @Value and @ConfigurationProperties
  • Command-line arguments

30. Different ways to package a Spring Boot application?

  • Executable JAR (default)
  • WAR file (for external servers)
  • Docker image
  • RPM/DEB using tools like Jib or Spring Boot Buildpacks

31. How do you secure a Spring Boot application?

  • Use Spring Security
  • Add spring-boot-starter-security
  • Configure custom login, roles, and access control
  • Use JWT, OAuth2, Basic Auth, etc.

32. Role of @EnableAutoConfiguration annotation?

It tells Spring Boot to automatically configure beans based on classpath contents, properties, etc.

33. What is the significance of @Entity?

  • Maps a Java class to a database table.
  • Used with JPA/Hibernate.
  • Each instance represents a row in the table.
@Entity
public class User {
   @Id
   private Long id;
   private String name;
}

34. Common Testing Annotations in Spring Boot

  • @SpringBootTest: Loads full application context
  • @WebMvcTest: For controller layer testing
  • @DataJpaTest: JPA/Hibernate testing
  • @MockBean: Add mocks to context
  • @TestConfiguration: Custom config for tests
