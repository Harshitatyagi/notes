Java Interview Questions

Q1. What is Java and why is it not considered a pure object-oriented programming language?

Answer:

Java is a high-level, class-based, object-oriented programming language designed to have as few implementation dependencies as possible. It was developed by Sun Microsystems and is now maintained by Oracle. Java follows the object-oriented paradigm closely but is not considered 100% object-oriented because it supports primitive data types (int, char, boolean, etc.) which are not objects.

Java uses wrapper classes (like Integer, Character, Boolean) to work with primitives in an object-oriented way, but this is optional. In pure object-oriented languages like Smalltalk, everything is an object.

Q2. What are the key components required to run a Java program?

Answer:

To run a Java program, the following components are needed:

  1. JDK (Java Development Kit): Contains tools to compile, debug, and run Java programs. Includes JRE.
  1. JRE (Java Runtime Environment): Contains JVM and libraries required to run applications.
  1. JVM (Java Virtual Machine): Interprets bytecode (.class file) and executes it on the host machine.
  1. .java file: Source code.
  1. .class file: Compiled bytecode by the Java compiler.

Q3. What are the main features of Java?

Answer:

  • Object-Oriented: Everything is organized as objects and classes.
  • Platform-Independent: "Write Once, Run Anywhere" using bytecode and JVM.
  • Simple & Familiar: Syntax is close to C++ but with less complexity.
  • Secure: Bytecode verification and sandbox execution.
  • Robust: Strong memory management, exception handling.
  • Multithreaded: Supports concurrent execution via threads.
  • High Performance: Through Just-In-Time (JIT) compiler.
  • Dynamic: Supports loading classes dynamically at runtime.

Q4. What is the Java String Pool?

Answer:

The String Pool is a special memory area in the heap where Java stores string literals.

When a string is created using double quotes, JVM checks the pool:

String s1 = "hello"; // Stored in the string pool
String s2 = "hello"; // Reuses s1

But when using new keyword:

String s3 = new String("hello"); // Stored in heap, not the pool

You can force pool usage using intern():

s3.intern();

Benefits: saves memory and improves performance.

Q5. What are Wrapper Classes in Java?

Answer:

Wrapper classes convert primitive types into objects. For example:

  • int → Integer
  • char → Character

Java supports autoboxing and unboxing:

Integer a = 10; // Autoboxing
int b = a; // Unboxing

Use cases:

  • Collections (like ArrayList) can only store objects.
  • Useful for type conversion and utilities.

Q7. What is the use of this and super keywords?

this: Refers to the current instance of a class. Used to:

  • Differentiate between instance variables and parameters.
  • Call one constructor from another (constructor chaining).
public class A {
   int x;
   A(int x) {
     this.x = x; // disambiguation
   }
}

super: Refers to the immediate parent class. Used to:

  • Access parent methods/constructors.
  • Call overridden methods.
public class B extends A {
   B() {
     super(5); // call parent constructor
   }
}

Q9. What are Constructors and their types?

Answer:

Constructors are special methods invoked at the time of object creation.

Types:

  1. Default Constructor: No parameters.
class A {
  A() { System.out.println("Default"); }
}
  1. Parameterized Constructor: With arguments.
A(int x) { this.x = x; }
  1. Copy Constructor: Manually create one to copy another object.
A(A obj) { this.x = obj.x; }

Java does not provide a default copy constructor like C++.

Q12. Method Overloading and Can we overload the main() method?

Answer:

Method Overloading means defining multiple methods with the same name but different parameter lists (type, number, or order).

void print(String s) {}
void print(int i) {}

Yes, we can overload the main() method, but only the signature public static void main(String[] args) is called by JVM. Others must be called manually.

public static void main(String[] args) {
    main(5);
}

public static void main(int x) {
    System.out.println("Overloaded main: " + x);
}

Q13. Method Overriding

Answer:

Method overriding is when a subclass provides a specific implementation for a method already defined in the parent class.

Rules:

  • Must have same method signature.
  • Access modifier must be same or more accessible.
  • Cannot override final, static, or private methods.
class Parent {
    void greet() {
        System.out.println("Hello from Parent");
    }
}
class Child extends Parent {
    @Override
    void greet() {
        System.out.println("Hello from Child");
    }
}

Q14. Exception Handling in Java

Answer:

Java uses a robust exception handling mechanism to handle runtime errors.

Keywords:

  • try – block where code might throw exception
  • catch – handles specific exception
  • finally – executes whether or not exception occurred
  • throw – used to explicitly throw an exception
  • throws – declares exceptions in method signature

Types:

  • Checked Exceptions: Handled at compile-time (IOException)
  • Unchecked Exceptions: Occur at runtime (NullPointerException)

Q15. Lifecycle of a Thread

Answer:

  1. New: Thread is created using new Thread().
  1. Runnable: Thread is eligible to run but not yet selected by the scheduler.
  1. Running: Thread is executing.
  1. Blocked/Waiting: Thread is waiting for resources or signal.
  1. Terminated: Thread has completed or aborted.
Thread t = new Thread();
t.start(); // Runnable -> Running

Q16. Singleton Class

Answer:

A Singleton class ensures only one instance is created and provides a global point of access.

Why use it?

  • Logging, caching, configuration classes.

Implementation:

public class Singleton {
    private static Singleton instance = null;
    private Singleton() {}
    public static Singleton getInstance() {
        if(instance == null) instance = new Singleton();
        return instance;
    }
}

For thread safety, use synchronized blocks or enum-based singleton.

Q18. Anonymous Inner Class

Answer:

An anonymous inner class is a class without a name and is declared and instantiated in a single expression.

Runnable r = new Runnable() {
    public void run() {
        System.out.println("Thread running");
    }
};
new Thread(r).start();

Use Case: GUI event handling, quick implementation of interface or abstract class methods.

Q19. Difference between Implicit and Explicit Type Conversion

Answer:

  • Implicit (Widening): Performed by the compiler.
int a = 10;
long b = a; // int to long
  • Explicit (Narrowing): Done manually using casting.
double d = 10.5;
int i = (int) d; // double to int

Explicit conversion may lead to data loss.

Q20. Purpose of volatile keyword

Answer:

The volatile keyword ensures visibility of changes to variables across threads.

Without volatile, threads may cache variable values and not read updates from main memory.

volatile boolean flag = true;

Used in multi-threaded programs to avoid stale data and maintain consistency.

Q23. Difference: final, finally, and finalize()

Answer:

  • final: Used to declare constants, prevent method overriding or inheritance.
final int x = 10;
  • finally: Always executes after try-catch. Used to release resources.
try {} catch(Exception e) {} finally { // clean-up }
  • finalize(): Method called by Garbage Collector before object is destroyed. Deprecated since Java 9.

Q24. WAP to toggle characters of a string

public String toggleCase(String input) {
    StringBuilder result = new StringBuilder();
    for (char c : input.toCharArray()) {
        if (Character.isUpperCase(c))
            result.append(Character.toLowerCase(c));
        else if (Character.isLowerCase(c))
            result.append(Character.toUpperCase(c));
        else
            result.append(c);
    }
    return result.toString();
}

Q25. WAP to count the occurrence of digits in a number

public void countDigits(int number) {
    int[] count = new int[10];
    while (number > 0) {
        count[number % 10]++;
        number /= 10;
    }
    for (int i = 0; i < 10; i++) {
        if (count[i] > 0)
            System.out.println("Digit " + i + ": " + count[i]);
    }
}

Q26. WAP to reverse an array

public void reverseArray(int[] arr) {
    int start = 0, end = arr.length - 1;
    while (start < end) {
        int temp = arr[start];
        arr[start] = arr[end];
        arr[end] = temp;
        start++;
        end--;
    }
}

Q27. Anagram Strings

public boolean isAnagram(String s1, String s2) {
    if (s1.length() != s2.length()) return false;
    char[] a1 = s1.toCharArray();
    char[] a2 = s2.toCharArray();
    Arrays.sort(a1);
    Arrays.sort(a2);
    return Arrays.equals(a1, a2);
}

Q28. WAP to find first and last occurrence in array

public int[] findFirstLast(int[] arr, int x) {
    int first = -1, last = -1;
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == x) {
            if (first == -1) first = i;
            last = i;
        }
    }
    return new int[] {first, last};
}

Q29. Why is String immutable in Java?

Answer:

  • Security: String is used in class loading and networking. Mutability may lead to security issues.
  • Thread-Safety: Immutable objects are thread-safe by default.
  • Caching: String Pool allows reuse, reducing memory footprint.
  • Hashcode caching: Immutable ensures consistent hashcode, which is vital for collections like HashMap.

Q30. Difference between Deep and Shallow Copy

Answer:

  • Shallow Copy: Creates a new object, but references the same inner objects.
Person p1 = new Person();
Person p2 = p1; // Shallow
  • Deep Copy: Creates a new object and also copies all nested objects.
Person p2 = new Person(p1); // Deep (requires clone or manual copy)

Use deep copy when mutable objects are involved to avoid side effects.
