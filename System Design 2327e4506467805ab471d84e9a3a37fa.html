✅ API Gateway – Complete Notes

📌 1. What is an API Gateway?

An API Gateway is a server that acts as the single entry point for all client requests in a microservices architecture. It handles various cross-cutting concerns, streamlining communication between clients and backend services.

🧠 2. Core Capabilities of API Gateway

CapabilityDescription
RoutingDirects requests to the appropriate backend service based on path, method, headers, etc.
Authentication & AuthorizationValidates user credentials and enforces access control using JWT, OAuth2, etc.
Rate Limiting & ThrottlingControls the number of requests a client can make in a given time to prevent abuse.
CachingCaches responses to improve performance and reduce backend load.
Request/Response TransformationModifies requests before forwarding or transforms responses before returning to clients.
Logging & MonitoringCollects metrics and logs for observability, debugging, and analytics.
API CompositionAggregates responses from multiple microservices into a single API response. Useful for mobile clients or composite UIs.
Protocol TranslationConverts between protocols (e.g., HTTP to WebSocket, REST to gRPC).
VersioningManages multiple versions of the same API to support backward compatibility.
Fault ToleranceImplements retries, circuit breakers, and fallback mechanisms to ensure resiliency.
SecurityProvides protections like IP whitelisting, TLS termination, and DoS defense.
Request ValidationEnsures input request parameters conform to schema or rules.

⚖️ 3. API Gateway vs Load Balancer

🚦 4. How API Gateway Handles Millions of Requests

Despite being a logical single point of entry, it can scale due to:

  • Horizontal scaling (multiple instances)
  • Auto-scaling in cloud environments
  • Load balancer in front (e.g., ALB)
  • Asynchronous I/O and event-driven architecture
  • Caching and rate limiting to reduce redundant load
  • Efficient service discovery behind the scenes

🌐 5. Route 53 – Brief Overview

AWS’s DNS service that:

  • Resolves domain names to IP addresses
  • Supports latency-based, weighted, and geo-based routing
  • Performs health checks to route only to healthy endpoints
  • Commonly used with ALB + API Gateway

🔁 6. Request Flow in Microservices Architecture

Typical architecture:

Client → DNS (Route 53)
      → External Load Balancer (ALB/NLB)
      → API Gateway
      → Internal Load Balancer (optional)
      → Microservices (via service discovery)

If using AWS-managed API Gateway:

Client → Route 53 → API Gateway → Lambda / VPC-based services
